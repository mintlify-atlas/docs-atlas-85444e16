TorchGeo samplers enable efficient sampling from geospatial datasets by returning geographic coordinates instead of integer indices.

## Base Classes

### GeoSampler

Abstract base class for sampling from `GeoDataset`. Unlike PyTorch's `Sampler`, `GeoSampler` returns geospatial information including latitude, longitude, height, width, projection, and time.

<ParamField path="dataset" type="GeoDataset" required>
  Dataset to index from
</ParamField>

<ParamField path="roi" type="Polygon | None" default="None">
  Region of interest to sample from (defaults to the bounds of dataset.index)
</ParamField>

<ParamField path="toi" type="pd.Interval | None" default="None">
  Time of interest to sample from (defaults to the bounds of dataset.index)
</ParamField>

## Single Samplers

### RandomGeoSampler

Samples elements from a region of interest randomly. Useful during training to maximize dataset size and return random chips. Note that randomly sampled chips may overlap.

<ParamField path="dataset" type="GeoDataset" required>
  Dataset to index from
</ParamField>

<ParamField path="size" type="tuple[float, float] | float" required>
  Dimensions of each patch. Can be a single float (used for both height and width) or a tuple of two floats (height, width)
</ParamField>

<ParamField path="length" type="int | None" default="None">
  Number of random samples to draw per epoch. Defaults to approximately the maximal number of non-overlapping chips that could be sampled
</ParamField>

<ParamField path="roi" type="Polygon | None" default="None">
  Region of interest to sample from
</ParamField>

<ParamField path="toi" type="pd.Interval | None" default="None">
  Time of interest to sample from
</ParamField>

<ParamField path="units" type="Units" default="Units.PIXELS">
  Defines if size is in pixel or CRS units
</ParamField>

<ParamField path="generator" type="Generator | None" default="None">
  Pseudo-random number generator (PRNG)
</ParamField>

### GridGeoSampler

Samples elements in a grid-like fashion. Useful during evaluation when making predictions for an entire region of interest while minimizing overlap between chips.

<ParamField path="dataset" type="GeoDataset" required>
  Dataset to index from
</ParamField>

<ParamField path="size" type="tuple[float, float] | float" required>
  Dimensions of each patch
</ParamField>

<ParamField path="stride" type="tuple[float, float] | float | None" default="None">
  Distance to skip between each patch (defaults to size). Usually slightly smaller than chip size to prevent stitching artifacts
</ParamField>

<ParamField path="roi" type="Polygon | None" default="None">
  Region of interest to sample from
</ParamField>

<ParamField path="toi" type="pd.Interval | None" default="None">
  Time of interest to sample from
</ParamField>

<ParamField path="units" type="Units" default="Units.PIXELS">
  Defines if size and stride are in pixel or CRS units
</ParamField>

### PreChippedGeoSampler

Samples entire files at a time. Useful for datasets that contain geospatial metadata and subclass `GeoDataset` but have already been pre-processed into chips.

<ParamField path="dataset" type="GeoDataset" required>
  Dataset to index from
</ParamField>

<ParamField path="roi" type="Polygon | None" default="None">
  Region of interest to sample from
</ParamField>

<ParamField path="toi" type="pd.Interval | None" default="None">
  Time of interest to sample from
</ParamField>

<ParamField path="shuffle" type="bool" default="False">
  If True, reshuffle data at every epoch
</ParamField>

<ParamField path="generator" type="Generator | None" default="None">
  Pseudo-random number generator used with shuffle
</ParamField>

## Batch Samplers

### BatchGeoSampler

Abstract base class for batch sampling from `GeoDataset`. Returns batches of geospatial information.

<ParamField path="dataset" type="GeoDataset" required>
  Dataset to index from
</ParamField>

<ParamField path="roi" type="Polygon | None" default="None">
  Region of interest to sample from
</ParamField>

<ParamField path="toi" type="pd.Interval | None" default="None">
  Time of interest to sample from
</ParamField>

### RandomBatchGeoSampler

Samples batches of elements from a region of interest randomly. Useful during training to maximize dataset size and return random chips.

<ParamField path="dataset" type="GeoDataset" required>
  Dataset to index from
</ParamField>

<ParamField path="size" type="tuple[float, float] | float" required>
  Dimensions of each patch
</ParamField>

<ParamField path="batch_size" type="int" required>

</ParamField>

<ParamField path="length" type="int | None" default="None">
  Number of samples per epoch. Defaults to approximately the maximal number of non-overlapping chips
</ParamField>

<ParamField path="roi" type="Polygon | None" default="None">
  Region of interest to sample from
</ParamField>

<ParamField path="toi" type="pd.Interval | None" default="None">
  Time of interest to sample from
</ParamField>

<ParamField path="units" type="Units" default="Units.PIXELS">
  Defines if size is in pixel or CRS units
</ParamField>

<ParamField path="generator" type="Generator | None" default="None">

## Units Enum

The `Units` enum defines the units of the `size` parameter:

```python
from torchgeo.samplers import Units

class Units(Enum):
    PIXELS = auto()  # Units in number of pixels
    CRS = auto()     # Units of coordinate reference system
```

## Usage Examples

### Random Sampling for Training

```python
from torch.utils.data import DataLoader
from torchgeo.datasets import Sentinel2
from torchgeo.samplers import RandomGeoSampler, Units

# Initialize dataset
dataset = Sentinel2(paths='s2_data/')

# Create random sampler
sampler = RandomGeoSampler(
    dataset=dataset,
    size=256,  # 256x256 pixel chips
    length=1000,  # 1000 samples per epoch
    units=Units.PIXELS
)

# Create dataloader
dataloader = DataLoader(
    dataset=dataset,
    sampler=sampler,
    batch_size=32,
    collate_fn=stack_samples
)
```

### Grid Sampling for Inference

```python
from torchgeo.datasets import Sentinel2
from torchgeo.samplers import GridGeoSampler, Units

dataset = Sentinel2(paths='s2_data/')

# Create grid sampler with overlap
sampler = GridGeoSampler(
    dataset=dataset,
    size=256,  # 256x256 pixel chips
    stride=224,  # 32 pixel overlap to prevent artifacts
    units=Units.PIXELS
)

dataloader = DataLoader(
    dataset=dataset,
    sampler=sampler,
    batch_size=1,
    collate_fn=stack_samples
)
```

### Region of Interest (ROI)

```python
from shapely.geometry import box
from torchgeo.samplers import RandomGeoSampler

# Define ROI as bounding box
roi = box(minx=-75.0, miny=40.0, maxx=-74.0, maxy=41.0)

sampler = RandomGeoSampler(
    dataset=dataset,
    size=256,
    roi=roi,  # Only sample from this region
    units=Units.PIXELS
)
```

### Time of Interest (TOI)

```python
import pandas as pd
from torchgeo.samplers import RandomGeoSampler

# Define time of interest
toi = pd.Interval(
    pd.Timestamp('2020-01-01'),
    pd.Timestamp('2020-12-31')
)

sampler = RandomGeoSampler(
    dataset=dataset,
    size=256,
    toi=toi,  # Only sample from this time period
    units=Units.PIXELS
)
```

### Pre-chipped Dataset

```python
from torchgeo.datasets import EuroSAT
from torchgeo.samplers import PreChippedGeoSampler

# For datasets already split into chips
dataset = EuroSAT(root='data', split='train')

sampler = PreChippedGeoSampler(
    dataset=dataset,
    shuffle=True  # Shuffle chips during training
)

dataloader = DataLoader(
    dataset=dataset,
    sampler=sampler,
    batch_size=32,
    collate_fn=stack_samples
)
```

## Key Concepts

- **Chips vs Tiles**: A chip is a small patch sampled from a larger tile. Samplers generate chip coordinates.
- **Overlap**: Grid samplers can create overlapping chips by setting `stride < size` to prevent stitching artifacts.
- **Units**: Specify whether dimensions are in pixels or CRS units (e.g., meters).
- **ROI/TOI**: Restrict sampling to specific geographic regions or time periods.
- **Geospatial indexing**: Samplers return `(xmin, xmax, ymin, ymax, tmin, tmax)` slices instead of integer indices.
