TorchGeo provides DataModules that integrate datasets, samplers, and data loaders for seamless training with PyTorch Lightning.

## Base Classes

### BaseDataModule

Base class for all TorchGeo data modules.

<ParamField path="dataset_class" type="type[Dataset]" required>
  Class used to instantiate a new dataset
</ParamField>

<ParamField path="batch_size" type="int" default="1">
  Size of each mini-batch
</ParamField>

<ParamField path="num_workers" type="int" default="0">
  Number of workers for parallel data loading
</ParamField>

<ParamField path="**kwargs" type="Any">
  Additional keyword arguments passed to dataset_class
</ParamField>

### GeoDataModule

Base class for data modules containing geospatial information.

<ParamField path="dataset_class" type="type[GeoDataset]" required>
  Class used to instantiate a new dataset
</ParamField>

<ParamField path="batch_size" type="int" default="1">
  Size of each mini-batch
</ParamField>

<ParamField path="patch_size" type="int | tuple[int, int]" default="64">
  Size of each patch, either size or (height, width)
</ParamField>

<ParamField path="length" type="int | None" default="None">
  Length of each training epoch
</ParamField>

<ParamField path="num_workers" type="int" default="0">
  Number of workers for parallel data loading
</ParamField>

<ParamField path="**kwargs" type="Any">
  Additional keyword arguments passed to dataset_class
</ParamField>

### NonGeoDataModule

Base class for data modules lacking geospatial information.

<ParamField path="dataset_class" type="type[NonGeoDataset]" required>
  Class used to instantiate a new dataset
</ParamField>

<ParamField path="batch_size" type="int" default="1">
  Size of each mini-batch
</ParamField>

<ParamField path="num_workers" type="int" default="0">
  Number of workers for parallel data loading
</ParamField>

<ParamField path="**kwargs" type="Any">
  Additional keyword arguments passed to dataset_class
</ParamField>

## Usage Example

```python
from lightning.pytorch import Trainer
from torchgeo.datamodules import EuroSATDataModule
from torchgeo.trainers import ClassificationTask

# Initialize datamodule
datamodule = EuroSATDataModule(
    root='data',
    batch_size=32,
    num_workers=4,
    download=True
)

# Initialize task
task = ClassificationTask(
    model='resnet50',
    in_channels=13,
    num_classes=10,
    lr=1e-3
)

# Train
trainer = Trainer(max_epochs=10)
trainer.fit(model=task, datamodule=datamodule)
```

## Custom DataModules

You can create custom DataModules by subclassing `GeoDataModule` or `NonGeoDataModule`:

```python
from torchgeo.datamodules import NonGeoDataModule
from torchgeo.datasets import RESISC45

class CustomDataModule(NonGeoDataModule):
    def __init__(self, root='data', batch_size=32, **kwargs):
        super().__init__(
            dataset_class=RESISC45,
            batch_size=batch_size,
            root=root,
            **kwargs
        )
```

## Key Features

- **Automatic splitting**: DataModules handle train/val/test splits
- **Integrated samplers**: GeoDataModules use geospatial samplers for efficient tiling
- **Data augmentation**: Built-in support for Kornia augmentations via `on_after_batch_transfer`
- **Multi-worker loading**: Configurable parallel data loading
- **Lightning integration**: Direct compatibility with PyTorch Lightning Trainer
