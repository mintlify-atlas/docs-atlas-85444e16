# Quickstart Guide

This guide will get you up and running with TorchGeo quickly through practical examples.

## Basic Imports

First, let's import the classes and functions we'll use:

```python
from lightning.pytorch import Trainer
from torch.utils.data import DataLoader

from torchgeo.datamodules import InriaAerialImageLabelingDataModule
from torchgeo.datasets import CDL, Landsat7, Landsat8, VHR10, stack_samples
from torchgeo.samplers import RandomGeoSampler
from torchgeo.trainers import SemanticSegmentationTask
```

## Example 1: Working with Geospatial Datasets

Geospatial datasets contain geographic metadata like coordinate reference systems (CRS) and spatial resolution. TorchGeo makes it easy to work with and combine these datasets.

<Steps>
  <Step title="Understanding the problem">
    Many remote sensing applications involve working with datasets that have:
    - Different coordinate reference systems (CRS)
    - Different spatial resolutions
    - Different numbers of spectral bands
    
    TorchGeo handles all of this complexity automatically!
  </Step>

  <Step title="Combine multiple satellite sources">
    Let's combine Landsat 7 and 8 imagery. Since Landsat 8 has more spectral bands, we'll use only the common bands:

    ```python
    # Create datasets for Landsat 7 and 8
    landsat7 = Landsat7(paths="...", bands=["B1", "B2", "B3", "B4", "B5", "B7"])
    landsat8 = Landsat8(paths="...", bands=["B2", "B3", "B4", "B5", "B6", "B7", "B8"])
    
    # Take the union - includes imagery from both satellites
    landsat = landsat7 | landsat8
    ```

    <Tip>
    The `|` operator creates a **UnionDataset** - queries work if data exists in *at least one* dataset.
    </Tip>
  </Step>

  <Step title="Add ground truth labels">
    Now let's add Cropland Data Layer (CDL) labels. We'll take the intersection to ensure we only sample regions with both imagery and labels:

    ```python
    # CDL supports automatic download and checksum verification
    cdl = CDL(paths="...", download=True, checksum=True)
    
    # Take the intersection - only sample where we have both Landsat AND CDL
    dataset = landsat & cdl
    ```

    <Tip>
    The `&` operator creates an **IntersectionDataset** - queries require data in *both* datasets.
    </Tip>
  </Step>

  <Step title="Sample image patches">
    Geospatial datasets often contain very large images (e.g., CDL covers the entire continental US!). Use samplers to extract patches:

    ```python
    # Random sampler: 256x256 pixel patches, 10,000 per epoch
    sampler = RandomGeoSampler(dataset, size=256, length=10000)
    
    # Create DataLoader with custom collation
    dataloader = DataLoader(
        dataset,
        batch_size=128,
        sampler=sampler,
        collate_fn=stack_samples
    )
    ```
  </Step>

  <Step title="Use in training">
    Now use the dataloader in your training loop:

    ```python
    for batch in dataloader:
        image = batch["image"]  # Landsat imagery
        mask = batch["mask"]    # CDL labels
        
        # Train a model, or make predictions using a pre-trained model
    ```
  </Step>
</Steps>

## Example 2: Benchmark Datasets for Object Detection

Benchmark datasets are similar to torchvision datasets - they include both images and labels for supervised learning tasks.

<Steps>
  <Step title="Initialize the dataset">
    Let's use the VHR-10 dataset for geospatial object detection:

    ```python
    from torch.utils.data import DataLoader
    from torchgeo.datamodules.utils import collate_fn_detection
    from torchgeo.datasets import VHR10
    
    # Download and extract automatically
    dataset = VHR10(root="...", download=True, checksum=True)
    ```

    <Note>
    Just like torchvision, TorchGeo handles downloading, checksumming, and extracting datasets automatically!
    </Note>
  </Step>

  <Step title="Create a DataLoader">
    Use the specialized collation function for detection tasks:

    ```python
    dataloader = DataLoader(
        dataset,
        batch_size=128,
        shuffle=True,
        num_workers=4,
        collate_fn=collate_fn_detection,
    )
    ```
  </Step>

  <Step title="Training loop">
    Each batch contains images, bounding boxes, labels, and masks:

    ```python
    for batch in dataloader:
        image = batch["image"]        # list of images
        bbox_xyxy = batch["bbox_xyxy"]  # list of bounding boxes
        label = batch["label"]        # list of class labels
        mask = batch["mask"]          # list of instance masks
        
        # Train a model, or make predictions using a pre-trained model
    ```
  </Step>
</Steps>

## Example 3: Training with Lightning

TorchGeo provides Lightning datamodules and trainers for reproducible research with minimal boilerplate.

<Steps>
  <Step title="Why Lightning?">
    Lightning integration provides:
    - Standard train/val/test splits
    - Pre-configured augmentations
    - Pre-calculated channel statistics
    - Reproducible results matching published papers
  </Step>

  <Step title="Create a datamodule">
    ```python
    datamodule = InriaAerialImageLabelingDataModule(
        root="...",
        batch_size=64,
        num_workers=6
    )
    ```
  </Step>

  <Step title="Configure a task">
    ```python
    task = SemanticSegmentationTask(
        model="unet",
        backbone="resnet50",
        weights=True,      # Use pre-trained weights
        in_channels=3,
        task="binary",     # Binary segmentation (building vs background)
        loss="bce",        # Binary cross-entropy loss
    )
    ```

    <Note>
    The `weights=True` parameter loads ImageNet pre-trained weights. For satellite-specific weights, see the pre-trained weights tutorial.
    </Note>
  </Step>

  <Step title="Train the model">
    ```python
    trainer = Trainer(default_root_dir="...")
    trainer.fit(model=task, datamodule=datamodule)
    ```

    That's it! Just four lines of code to train a semantic segmentation model.
  </Step>
</Steps>

## Using Pre-trained Weights

TorchGeo is the first library to provide models pre-trained on multispectral satellite imagery:

```python
import timm
from torchgeo.models import ResNet18_Weights

# Load Sentinel-2 pre-trained weights
weights = ResNet18_Weights.SENTINEL2_ALL_MOCO

# Create model with correct number of input channels
model = timm.create_model(
    "resnet18",
    in_chans=weights.meta["in_chans"],
    num_classes=10
)

# Load pre-trained weights
model.load_state_dict(weights.get_state_dict(progress=True), strict=False)
```

<Tip>
Pre-trained weights can also be used directly in Lightning tasks via the `weights` argument!
</Tip>

## Command-Line Interface

TorchGeo also supports training via CLI using LightningCLI:

<Steps>
  <Step title="Create a config file">
    ```yaml config.yaml
    trainer:
      max_epochs: 20
    model:
      class_path: ClassificationTask
      init_args:
        model: 'resnet18'
        in_channels: 13
        num_classes: 10
    data:
      class_path: EuroSAT100DataModule
      init_args:
        batch_size: 8
      dict_kwargs:
        download: true
    ```
  </Step>

  <Step title="Run training">
    ```bash
    # Train and validate
    torchgeo fit --config config.yaml
    
    # Validate only
    torchgeo validate --config config.yaml --ckpt_path=...
    
    # Test
    torchgeo test --config config.yaml --ckpt_path=...
    ```
  </Step>

  <Step title="Get help">
    ```bash
    # See available stages
    torchgeo --help
    
    # See trainer options
    torchgeo fit --help
    
    # See model options
    torchgeo fit --model.help ClassificationTask
    
    # See data options
    torchgeo fit --data.help EuroSAT100DataModule
    ```
  </Step>
</Steps>
